# GY-NEXUS 쇼핑 쇼츠 내부검색 V2.2

## 완성된 기능

`/admin/shopping-shorts` 안에서 상품명 또는 중국어 간체 키워드를 입력하면 도우인·샤오홍슈의 공개 웹 색인 콘텐츠 후보를 카드로 보여줍니다.

- 도우인 + 샤오홍슈 통합 검색
- 도우인만 또는 샤오홍슈만 선택 검색
- 최대 12개 결과 카드
- 대표 이미지, 플랫폼, 제목, 원문 출처 표시
- 여러 카드 선택
- 선택 카드를 현재 프로젝트의 AI 소스함에 즉시 저장
- 중복 URL 차단과 소스함 20개 제한
- 검색 카드 대표 이미지를 AI 믹스의 시각 참고자료로 전달
- 기존 `선택 소스로 AI 짜집기 설계`와 바로 연결
- 반복 검색 5분 캐시와 과도한 호출 제한

## 대표 사용 순서

1. 관리자에서 `쇼핑 쇼츠 센터`를 엽니다.
2. 작업할 상품 프로젝트를 선택합니다. 프로젝트가 없다면 상품명·확인된 설명·상품 사진으로 먼저 만듭니다.
3. `내 사이트에서 도우인·샤오홍슈 찾기`에 상품명 또는 중국어 간체 검색어를 입력합니다.
4. `도우인 + 샤오홍슈`, `도우인만`, `샤오홍슈만` 중 범위를 고릅니다.
5. `GY 내부에서 검색`을 누릅니다.
6. 참고할 카드 여러 개를 선택하고 `AI 소스함에 담기`를 누릅니다.
7. 아래 소스함에서 `AI 믹스 선택` 상태와 권리 상태를 확인합니다.
8. 더 깊은 장면 분석이 필요하면 해당 자료의 대표 이미지나 직접 캡처한 핵심 프레임을 이용해 `AI 장면 분석`을 실행합니다.
9. `선택 소스로 AI 짜집기 설계`를 누르면 15~30초 한국형 판매 순서가 만들어집니다.
10. 대본·자막·음성·장면 생성·품질 승인·MP4 렌더 순서로 완성합니다.

## 검색의 정확한 범위

이번 기능은 도우인·샤오홍슈의 비공개 API나 로그인 쿠키를 긁는 방식이 아닙니다. OpenAI의 웹 검색 도구가 공개 웹에 색인된 원문 페이지를 도메인 제한으로 찾고, GY-NEXUS 안에 출처 링크가 있는 카드로 표시합니다.

따라서 플랫폼 로그인 뒤에만 보이는 콘텐츠, 색인되지 않은 최신 영상, 실시간 조회수 순위는 검색되지 않을 수 있습니다. 결과가 부족하면 중국어 간체 키워드로 다시 검색하거나 `원문 검색` 보조 링크와 직접 링크 추가 기능을 사용합니다.

도우인에는 승인된 앱을 위한 공식 키워드 영상 검색 API도 있지만, 앱 권한과 관리 콘솔의 키워드 등록 등 별도 조건이 필요합니다. 대표님이 심사를 미룬 현재 V2.2는 그 API를 가장하지 않고 공개 웹 검색을 사용합니다. 나중에 공식 승인이 끝나면 같은 카드 화면의 검색 공급자를 공식 API로 교체할 수 있게 분리해 두었습니다.

## 저작권과 최종 영상 사용

- 내부 검색 카드는 `권리 미확인 · 페이지 링크`로 저장됩니다.
- AI는 카드 제목·대표 이미지·대표가 적은 메모에서 훅, 촬영각도, 사용 상황, 판매 구조를 참고합니다.
- 권리 미확인 원본 영상의 자동 다운로드·워터마크 제거·좌우 반전·무단 재편집은 하지 않습니다.
- 권리 미확인 카드는 최종 원본 컷 사용이 서버에서 차단됩니다.
- 직접 촬영, 판매자 제공, 제휴센터 제공, 사용 허가 영상만 파일로 올려 최종 컷으로 쓸 수 있습니다.
- 최종 영상은 참고 구조를 바탕으로 상품 사진·허가 영상·새 AI 장면을 사용해 새 순서로 재구성합니다.

## 환경변수

기존 V2.1 배포에 `OPENAI_API_KEY`가 이미 설정되어 있으면 내부 검색은 바로 사용할 수 있습니다.

선택 환경변수:

```env
OPENAI_WEB_SEARCH_MODEL=gpt-5.6
```

값을 추가하지 않으면 `gpt-5.6`을 기본으로 사용합니다. 도우인·샤오홍슈 공식 앱 심사는 이 공개 웹 내부검색 기능에 필요하지 않습니다. 공식 키워드 검색, 계정 업로드, 본인 영상 관리 등 플랫폼 OpenAPI 기능을 나중에 연결할 때는 해당 앱 키·권한·사용자 승인이 필요합니다.

## 설치 후 확인

1. 변경 파일을 Git에 커밋하고 `main`으로 푸시합니다.
2. Vercel 배포가 `Ready`가 될 때까지 기다립니다.
3. `https://gy-nexus-zfpq.vercel.app/admin/shopping-shorts`에 로그인합니다.
4. 내부검색 버튼과 카드 선택 화면이 보이는지 확인합니다.
5. 첫 실제 검색은 배포 환경의 `OPENAI_API_KEY`가 있어야 작동합니다.
