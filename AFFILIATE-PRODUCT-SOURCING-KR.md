# GY-NEXUS 인기상품·제휴 링크 연동 보완판

## 이번 보완의 기준

연결 카드의 초록색 표시보다 실제 상품 데이터가 들어오고, 제휴 링크가 보존되며, 실패 이유를 확인할 수 있는지를 우선했습니다.

- 쿠팡: API 권한이 있는 계정에서 골드박스, 카테고리 베스트, 키워드 검색을 실제 호출합니다.
- Temu: 공개 Affiliate API가 없는 계정에 Seller API나 임의의 링크 템플릿을 요구하지 않습니다. Affiliate 대시보드에서 `공유`로 만든 링크를 등록합니다.
- 공통: URL뿐 아니라 쿠팡·Temu 상품 ID 기준으로 중복을 제거하고 데이터 완성도, 링크 근거, 마지막 확인 시간, 실행 결과를 저장합니다.
- 공통: 후보 수집만으로 자동 게시하지 않습니다. Product Intelligence에서 대표 승인을 거쳐야 정식 상품이 됩니다.

## 적용 순서

1. Supabase Dashboard의 SQL Editor에서 `supabase/AFFILIATE-PRODUCT-SOURCING-MIGRATION.sql` 전체를 실행합니다.
2. 이 ZIP을 기존 GitHub 프로젝트 폴더에 전체 덮어씁니다.
3. 쿠팡 API 권한이 있는 경우 Vercel에 `COUPANG_ACCESS_KEY`, `COUPANG_SECRET_KEY`를 등록합니다.
4. 선택 사항으로 쿠팡 채널 추적 ID가 있으면 `COUPANG_SUB_ID`를 등록합니다.
5. Temu는 `TEMU_AFFILIATE_ID`가 없어도 공유 링크를 저장해 운영할 수 있습니다. ID는 계정 식별용 선택값입니다.
6. GitHub에 push한 뒤 Vercel 배포가 끝나면 `/admin/product-intelligence`를 엽니다.

## 쿠팡 사용법

`Product Intelligence → 쿠팡 자동 수집`에서 아래 중 하나를 선택합니다.

- 오늘의 특가: 골드박스 상품
- 카테고리 베스트: 가전디지털, 생활용품, 주방용품 등
- 키워드 검색: 세탁조 클리너, 태블릿, USB-C 허브 등

`인기상품 불러오기 · 중복 정리 · 가치 분석`을 누르면 쿠팡 API의 HMAC 서명, 권한, 응답 상품, 제휴 링크를 실제로 확인한 뒤 후보 목록에 저장합니다.

## Temu 사용법

1. Temu Affiliate Program 대시보드에서 인기상품 또는 상품검색을 엽니다.
2. 홍보할 상품의 `공유` 버튼을 눌러 제휴 링크를 복사합니다.
3. `Product Intelligence → Temu 공유 링크`에 한 줄에 한 상품씩 붙여넣습니다.

기본 형식:

```text
https://Temu에서_복사한_공유링크
```

상품몰이 자동 정보 읽기를 막은 경우:

```text
https://Temu에서_복사한_공유링크 | 상품명 | 19,900원
```

최대 10개를 한 번에 처리하며, 잘못된 링크는 전체 작업을 중단시키지 않고 제외 이유를 화면에 표시합니다.

## 화면 상태의 정확한 뜻

- `연결 완료`: 외부 API의 서명·권한·상품 응답을 최근 24시간 안에 실제 확인함
- `링크 운영 준비`: 공개 API 방식은 아니지만 검증된 공유 링크가 저장되어 실제 운영 가능함
- `승인·검증 필요`: 키나 계정 정보는 있지만 실제 응답 확인이 끝나지 않음
- `설정 필요`: 자동 API 호출에 필요한 값이 없음

## 중요한 제한

쿠팡 API 키는 쿠팡 파트너스에서 해당 계정에 API 권한이 발급되어야 사용할 수 있습니다. Temu Seller Open Platform은 판매자 운영 API이며 Affiliate 인기상품 수집용 공개 API로 간주하지 않습니다. GY-NEXUS는 접근 제한을 우회하거나 대시보드를 무단 크롤링하지 않습니다.
