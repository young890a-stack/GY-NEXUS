# GY-NEXUS AI Company OS v2.0 — Master Blueprint

## 제품 원칙

**One Reliable Pipeline. Real Business Results.**

기능 수를 늘리는 대신 다음 한 흐름을 반복 가능하고 추적 가능하게 완성한다.

상품 후보 수집 → 사실 검증 → 기회 분석 → 대표 승인 → 블로그·쇼츠·썸네일 생성 → 품질 검수 → 예약 게시 → 성과 분석 → 다음 추천

## v2.0 첫 완성 기준

쿠팡 또는 Temu 상품 1개가 검증 가능한 방식으로 입력되고, 상품 스냅샷이 저장되며, AI 분석 결과와 근거가 표시되고, 대표 승인 후 블로그·쇼츠·썸네일 패키지가 만들어지고, 품질 검수를 통과한 작업만 Blogger 예약 게시 큐에 들어가야 한다. 실패 시 단계와 원인을 확인하고 재실행할 수 있어야 한다.

## 썸네일 판단 엔진

AI는 카테고리, 타깃, 플랫폼, 제목 의도, 상품 이미지 품질을 판단해 콘셉트·색상·구도·문구·비율을 선택한다. 생성 후 아래 조건을 통과해야 한다.

- 상품과 이미지 일치
- 최신 확인 없는 가격·할인율 금지
- 모바일 가독성
- 핵심 메시지 하나
- 과장·허위 표현 금지
- 플랫폼 비율 적합

90점 미만을 무조건 재생성하는 방식 대신, 실패한 항목을 특정하여 수정 생성한다. 점수는 참고 지표이며 실제 CTR 예측값으로 표시하지 않는다.

## 데이터 신뢰 등급

- VERIFIED: 공식 API 또는 대표가 확인한 원본
- PARTIAL: 일부 필드만 검증됨
- UNVERIFIED: 출처·가격·상세정보 확인 필요

UNVERIFIED 상품은 자동 게시하지 않는다.

## 개발 순서

1. Sprint 1 — 수동/공식 입력 상품 1개 → 근거 기반 분석·썸네일 계획
2. Sprint 2 — 콘텐츠 패키지 생성·저장·품질 게이트
3. Sprint 3 — Blogger 초안/예약 게시 실제 연결
4. Sprint 4 — 클릭 추적·성과 회수·학습 리포트
5. Sprint 5 — 쿠팡 공식 제공 범위 연결
6. Sprint 6 — Temu 공식 제공 범위 연결
7. Sprint 7 — 안정화 후 승인 정책별 자동화

## 금지 원칙

- 공식 데이터가 아닌데 실시간 인기순위라고 표시하지 않는다.
- 가격·할인·판매량을 추측하지 않는다.
- 연결되지 않은 기능을 작동 중으로 보이지 않는다.
- 외부 게시를 대표 승인 없이 기본 실행하지 않는다.
- 데이터베이스 마이그레이션 없이 운영 데이터를 변경하지 않는다.
